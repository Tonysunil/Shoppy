# Project Documentation Rules
When writing or generating content for the MyShoppy project documentation, STRICTLY adhere to the following formatting guidelines as requested by the user:

1. **Font Type:** Times New Roman for ALL text.
2. **Main Headings:** Size 16, Bold, Left Alignment, UPPER CASE.
3. **Sub-headings:** Size 14, Bold, Left Alignment, UPPER CASE.
4. **Sub-sub-headings:** Size 12, Bold, Left Alignment, UPPER CASE.
5. **Text Font Size:** Size 12.
6. **No Underlining:** Do not underline any headings.
7. **Figure Labels:** ALWAYS give a number and name for figures (e.g., "Fig: 3.1 Data Model").
8. **Table Labels:** ALWAYS give a number and name for tables (e.g., "Table: 4.1 Employee").
9. **Line Spacing:** 1.5" spacing for all text.
10. **Paragraph Integrity:** Do not split paragraphs across two pages.
11. **Page Numbering Exclusions:** Do NOT give page numbers for the first page, certificate page, index page, list of figures page, list of tables page, abstract, and each chapter heading page.
12. **Header Requirements:** Every page (except the exclusions in rule 11) must contain the Project Title at the top-left corner and the Page Number at the top-right corner.
13. **Heading Weight:** ALL headings must be bold.
14. **Chapter Pagination:** Start every new chapter on a fresh/new page.
15. **Page Setup:** 
    - Margins: Top 1", Bottom 1", Left 1.25", Right 1"
    - Paper: Width 8.27", Height 11.69" (A4 paper), Portrait orientation.
